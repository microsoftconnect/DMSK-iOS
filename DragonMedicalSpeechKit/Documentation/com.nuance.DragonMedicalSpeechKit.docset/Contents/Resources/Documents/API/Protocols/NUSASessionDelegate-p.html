DragonMedicalSpeechKit
Available in version 1.0 and later
NUSASessionDelegate.h

Overview

NUSASession delegate messages.

Delegate messages sent by the NUSASession class to its delegate receiver. All messages provided by the protocol are optional and can be implemented by the receiver.

Instance methods

sessionDidStartRecording

Sent after audio data recording has started.
- (void) sessionDidStartRecording
Discussion

Sent to the delegate whenever a recording session is started.

Availability
Available in version 1.0 and later.

sessionDidStopRecording

Sent after audio data recording has stopped and standby mode disabled.
- (void) sessionDidStopRecording
Discussion

Availability
Available in version 3..9 Sent to the delegate in the following conditions: Standby mode is disabled and the recording session is stopped. An error occurs when starting the recording session. An error occurs starting standby mode. Long press on the microphone button of the SpeechBar when the microphone is in recording or standby mode. and later.

sessionDidStartSpeaking

Sent after speaking has started.
- (void) sessionDidStartSpeaking
Discussion

Sent to the delegate whenever speaking is started. If a call to startSpeaking() cancels a previous call to startSpeaking(), this message is not sent again.

Availability
Available in version 1.3 and later.

sessionDidStopSpeaking

Sent after speaking has stopped.
- (void) sessionDidStopSpeaking
Discussion

Sent to the delegate whenever speaking is finished. This message is sent when the session has finished speaking the provided text or stopSpeaking() is called. If speaking is cancelled by another call to startSpeaking(), this message is only sent after the last text has been finished.

Availability
Available in version 1.3 and later.

sessionDidStartStandby

Sent after standby mode has started.
- (void) sessionDidStartStandby
Discussion

Availability
Available in version 3.9 Sent to the delegate in the following conditions: 1.Standby mode enabled and started. 2.Standby mode is enabled and the recording session is stopped. and later.

sessionDidReceiveInvalidAuthenticationToken

Sent after authentication token is invalid.
- (void) sessionDidReceiveInvalidAuthenticationToken
Discussion

Sent to the delegate whenever the authentication token is been considered invalid. Once this message is sent the user must get new token from authentication library and send the new token by calling updateAuthenticationToken API

Availability
Available in version 6.0 and later.

sessionShouldContinueRecordingDuringCallBanners

Configures interrupt handling behavior when system displays incoming calls using a banner.
- (BOOL) sessionShouldContinueRecordingDuringCallBanners
Return Value

Return true to continue recording. By default SDK stops recording on receiving incoming call or other audio interrupts
