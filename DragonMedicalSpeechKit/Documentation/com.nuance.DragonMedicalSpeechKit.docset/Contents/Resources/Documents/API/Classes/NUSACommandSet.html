    Overview

    Representation of an application-defined command set.

    A command set is a group of application commands that logically belong together. It is characterized by a human-readable name and a description that will appear in the "What You Can Say" page.

    Command sets contain a set of application commands. Each command must have a unique ID that identifies the recognized command in the vuiControllerDidRecognizeCommand() delegate message. Commands should have a human-readable description and a spoken form. CommandPlaceholders can be used in spoken forms.

    A command set can be enabled (default) or disabled, in which case all the commands it contains are disabled.

    Properties

    isEnabled

    A Boolean value that determines whether a command set is enabled.
    @property BOOLisEnabled
    Discussion

    When set to YES, the command set is enabled, else disabled. The default value of this property is YES.

    Availability
    Available in version 1.3 and later.

    createCommand:phrase:displayString:description:

    Adds a new application command to the command set.
    - (void) createCommand: (NSString *)id  phrase: (NSString *)phrase  displayString: (NSString *_Nullable)displayString  description: (NSString *)description 
    Parameters
    id

    The unique identifier of the command. Make sure the application command ID does not contain spaces and does not start with a number.

    phrase

    The application command phrase.

    displayString

    The application command display string (optional).

    description

    The human-readable description of the command. Can contain Placeholder identifiers: in this case, the corresponding placeholder objects must be created and supplied to the VuiController in a assignCommandPlaceholders() method call.

    Discussion

    The application must provide a unique id, a description and the command phrase. To create alternative phrases for a command call this method multiple times for the same id.

    Important: Raises an NSInvalidArgumentException if the command id or phrase is nil or empty. Placeholder identifiers must be enclosed in < and > in the spoken form. For example, if the ID of the referenced placeholder is "patient", a correct spoken form can be "select patient <patient>"

    Alternative spoken forms are created by calling this method multiple times for the same ID.

    Availability
    Available in version 1.3 and later.
