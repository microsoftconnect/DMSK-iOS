Dragon Medical Virtual Assistant entity management

Entities help users experience high recognition accuracy; each entity has a set of values and entity management is used to define the entity values for entities such as RECEPIENT_DYNAMIC, NOTE_DYNAMIC, etc. The data uploaded to dynamic entities doesn't persist in the DMVA Server after the session is closed. The data can be removed mid-session by requesting it to be cleared.

These requests don't have any effect on the DMVA states. These requests can be made any time when the DMVA session is open and will be processed in the background. Dynamic entities are compiled into the grammar model and can be used for larger lists.
