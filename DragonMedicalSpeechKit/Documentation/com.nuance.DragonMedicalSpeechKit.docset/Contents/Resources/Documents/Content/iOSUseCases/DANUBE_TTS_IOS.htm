Text-To-Speech (TTS)

Note: Only US English is currently supported.

Prerequisites

You've speech-enabled your app.

Procedure

Enabling TTS

To subscribe to notifications on TTS playback, do the following:

1. Create and instantiate a class that implements the NUSASessionDelegate, including the methods sessionDidStartSpeaking and sessionDidStopSpeaking.

2. Call [[NUSASession sharedSession] setDelegate: @"text"].

Working with TTS

To start speaking text, send the startSpeaking: message to the shared session instance:

[[NUSASession sharedSession] startSpeaking:@"text"];

text â€“ the text to be spoken.

To stop before the complete text has been spoken, send the stopSpeaking: message to the shared session instance.

Remarks

  • If speaking is already active when startSpeaking: is called, new requests are queued. In this case, the sessionDidStopSpeaking delegate message is sent only when all queued requests have been completely spoken.
  • If recording is active when speaking is started, recording is paused until speaking has finished. No recording-related session delegate messages are sent when recording is paused and resumed.
  • If startRecording: is called while speaking is active, a sessionDidStartRecording message is sent to the delegate immediately but recording is paused until speaking is finished.

Dragon Medical Virtual Assistant

Note: DMVA is only available in US Nuance-hosted systems and requires an additional license.

DMVA provides TTS prompts which you can display as text on the user interface and play as audio if the user hasn't provided enough information to finish the intent (the action for the iOS app to perform). You can use the predefined TTS prompts from the DMVA dialog results or define your own.

For more information on DMVA, see: Dragon Medical Virtual Assistant - overview.
