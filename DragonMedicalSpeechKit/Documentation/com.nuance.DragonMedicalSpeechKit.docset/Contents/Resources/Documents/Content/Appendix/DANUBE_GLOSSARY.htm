Glossary

A

Android edition Edition of Dragon Medical SpeechKit designed for Android devices; for example, Samsung Galaxy Tab.

audio device A device which is connected to a computer to record and/or play back sound; for example, a microphone.

B

Browser edition Edition of Dragon Medical SpeechKit designed for web browser apps.

C

COM edition Edition of Dragon Medical SpeechKit designed for COM desktop apps.

command See: voice command.

control device A microphone (such as Philips SpeechMike or Nuance PowerMic) or foot pedal that provides (configurable) buttons for starting/stopping recording, etc.

correction Editing text to replace incorrectly recognized words or phrases.

D

 

E

 

F

field An end user text control in your app that can be speech-enabled. When the end user speaks, the recognized text is written into the field with the speech focus.

field navigation Moving between text controls using voice commands or by tapping.

frontend recognition The text is recognized in real time on the GUI. End users can correct recognition errors immediately after or during a recording session. See also: speech recognition.

G

 

H

 

I

Insert mode Recording mode which inserts new speech at the current cursor position in the sound file but doesn't overwrite existing parts of the sound file.

iOS edition Edition of Dragon Medical SpeechKit designed for iOS devices; for example, iPhone, iPad.

J

 

K

KPI (Key Performance Indicator) Data about the speech recognition features that are used by the end user and how they're used. This data is collected, analyzed and used to tailor the product to real-life use cases.

L

license See: usage-based license.

M

medical specialty See: topic.

N

.NET edition Edition of Dragon Medical SpeechKit designed for .NET desktop apps.

Nuance Order Desk Service for ordering licenses.

Nuance partner Company who integrates Dragon Medical SpeechKit into its app.

O

Overwrite mode Recording mode which places new speech at the current cursor position in the sound file and overwrites existing speech.

P

partner See: Nuance partner.

Q

 

R

recognition result All sounds recognized by the speech recognition engine, including dictated speech (which is converted into text), redundant speech and background noise (which are ignored), as well as voice commands (which are executed).

S

scratch that Voice command that deletes the most recently dictated text up to the last punctuation mark. The maximum length of the text that can be deleted using the scratch that command is approximately 3 minutes or 200 characters.

speech bar A GUI element providing the end user with visual feedback on recording.

speech-enabled control GUI text control (field) that can accept spoken input and convert the sound to text.

speech focus Refers to the active speech-enabled text control (field) that will accept spoken input and display the recognized text.

speech recognition Computerized recognition of spoken words. Speech is transformed into text, this means written out or interpreted as voice commands to which the program responds with the requested action. See also: frontend recognition.

speech recognition language The language used for speech recognition.

T

tap to navigate To move the speech focus to another speech-enabled control with the tap gesture.

topic The medical specialty end users work in.

U

usage-based license A license scheme based on the amount of sound data processed by speech recognition.

V

VA level Voice-activated recording level, also referred to as silence detection level, voice sensitivity threshold or VOR level. The VA level is a threshold level of the recording volume which allows the recording app to distinguish between speech (which should be recorded) and silence or background noise (which shouldn't be recorded). Recording stops automatically (after about 20 seconds) when the speech input volume is below the VA level.

voice command A spoken word or phrase that initiates an action in an app.

VUI Voice User Interface.

VUI bundle A custom set of voice commands that usually correspond to fields in a form. It can be seen as the VUI equivalent of a GUI view.
