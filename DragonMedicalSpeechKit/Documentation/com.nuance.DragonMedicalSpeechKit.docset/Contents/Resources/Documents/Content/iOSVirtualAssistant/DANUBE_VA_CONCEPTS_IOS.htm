Terminology and states

Dragon Medical Virtual Assistant (DMVA) enables a 'dialog' between your iOS app and the clinician, including speech, text and commands for your app to perform.

During this dialog, DMVA uses strings recognized from what the clinician says to determine the commands.

A DMVA command consists of an intent and one or more concepts:

  • Intent: The action for the app to perform (for example, order a lab test or schedule an appointment).
  • Concept: The variable part of an intent (for example: the patient's name, dates, test types, department names).

Dynamic concepts

Dynamic concepts are concepts that can be modified during a DMVA session, for example, the list of clinicians.

To modify dynamic concepts during a session, you can do either of the following:

  • Upload concept values to the DMVA server via the uploadValuesForConcept method. Uploaded concept values remain available until clearUploadedValuesForConcept is called. Concept values are uploaded asynchronously.
  • Pass concept values as inline parameters via the setInlineValuesForConcept method, for immediate use. Inline concept values are cleared automatically when the DMVA session is closed.

Important: Don't call uploadValuesForConcept and setInlineValuesForConcept on the same concept.

DMVA states and their transitions

DMVA can be in one of the following states:

  • Closed - NUSAVirtualAssistantStateClosed
  • Open - NUSAVirtualAssistantStateOpened

To transition from closed to open, call the openWithModel method and specify a grammar model. DMVA performs the required license and model verification before transitioning to the open state. When DMVA transitions to the open state, the NUSAVirtualAssistantControllerDelegateâ€™s stateChanged method is triggered, with the state parameter set to Opened.

From the open state, call close to discard any work in progress and transition DMVA to the closed state. When DMVA transitions to closed, the stateChanged event is triggered, with the state parameter set to Closed.
