Release Notes for Dragon Medical SpeechKit 3.3 Release 2 (iOS edition)

New in version 3.3 Release 2

  • For users with a Dragon Medical One license, if text is selected via voice, a correction menu is now displayed; this menu offers alternative speech recognition results and additional functionality. [DNB-6926]

Solved issues

  • Dictation results are now inserted in the correct text field after the user has closed a notification. [DNB-11305]
  • Debugging symbols have been removed from the Dragon Medical SpeechKit embedded framework. [DNB-11579]
  • UISearchBar text controls can now be disabled. [DNB-10991]

Note: For information on previous releases, see: Release history.

Important information

  • From version 3.1 Service Release 12, the iOS edition is delivered as an embedded framework. When upgrading from a version earlier than this, changes are required in your integration. For more information, see 'Setting up Dragon Medical SpeechKit' in the Dragon Medical SpeechKit iOS edition help. [DNB-8963, DNB-10129]
  • The SystemConfiguration framework must be linked explicitly to your application, even if Link Frameworks Automatically is set to YES.
  • If Link Frameworks Automatically is set to NO, the WebKit framework must also be linked to your application.
  • Dragon Medical SpeechKit 3.3 (iOS edition) is compatible with server versions 3.1, 3.2 and 3.3.
  • Dragon Medical Server 3.3 is compatible with client versions 3.1, 3.2 and 3.3.

Known issues

  • A 120-byte memory leak occurs in the SDK each time the personalization & help screen is opened. [DNB-11775]
  • A 16-byte memory leak occurs in the SDK each time a session is closed. [DNB-5441]
  • A 78-byte memory leak occurs in the SDK each time the correction menu is displayed. [DNB-12363]
  • iPhones on iOS 8: The Manage Vocabulary and Manage Auto-text pages are not displayed properly if rotated to landscape mode while visible; this is due to a known issue in iOS 8. Solution: Upgrade to iOS 9, 10 or 11. [DNB-8483]

Infrastructure

Infrastructure no longer supported

  • Dragon Medical SpeechKit and Dragon Medical embedded 3.3 applications do not work with version 2.1 servers.
  • Version 2.1 client applications do not work with version 3.3 servers.

Infrastructure changes planned for the next release

  • Dragon Medical SpeechKit and Dragon Medical embedded 3.4 applications will not work with version 3.0 servers.
  • Version 3.0 client applications will not work with version 3.4 servers.
  • Dragon Medical SpeechKit and Dragon Medical embedded 3.4 applications will not work with iOS 8 devices.

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third party product and supporting Nuance documentation.

When support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s end-of-life and Nuance's policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.

Dragon Medical SpeechKit product life cycle

Nuance provides support throughout the life cycle of one version and point release. Please note that Nuance will not perform problem analysis after the scheduled start of restricted maintenance. Dragon Medical Server versions released over three months after the dates stated here will not work with these client editions. For up-to-date information on these schedules, go to Nuance iSupport.

Release history

New in version 3.2 Release 6

  • iOS 11 is now supported. [DNB-11563]
  • Xcode 9 is now supported. [DNB-11614]

Solved in version 3.2 Release 6

  • The app no longer becomes unresponsive when recording a long dictation (several hours). [DNB-10697]
  • Switching between multiple speech-enabled forms while recording no longer stops the recording. [DNB-11081]
  • An irregularly-occurring 80-byte memory leak has been resolved. [DNB-11714]
  • iOS 11: The personalization & help screen no longer makes the app stop unexpectedly. [DNB-11659]
  • iOS 11: The volume meter is now displayed above the keyboard (y axis) instead of overlaying the keyboard (z order). This change is relevant if the volume meter is visible and displayed at the bottom of the screen (NUSAVolumeMeterLocationBottom; this is the default when the speech control is enabled) or on the sides of the screen (NUSAVolumeMeterLocationSide). [DNB-11572]
