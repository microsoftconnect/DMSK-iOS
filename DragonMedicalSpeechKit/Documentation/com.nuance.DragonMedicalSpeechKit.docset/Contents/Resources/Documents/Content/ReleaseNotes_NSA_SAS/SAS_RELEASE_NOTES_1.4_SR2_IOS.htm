Release Notes for 360 | SpeechAnywhere Services 1.4 Service Release 2 (iOS edition)

Solved issues

The application no longer shuts down unexpectedly after removing and releasing the active control and then calling synchronize(). [PSP00069377]

Important information

  • 360 | SpeechAnywhere Services 1.4 Server is compatible with client versions 1.2 and 1.3.
  • The delete field voice command has been removed.
  • In SAS "license GUID" has changed to "organization token" and in NMC "customer token" has changed to "organization token".
  • GUI views that are hidden or disabled or whose editable or userInteractionEnabled properties are set to NO are now excluded from speech recognition by default. For information on how to include such GUI views in speech recognition, see: "Including hidden or disabled text controls in speech recognition" in the SDK Help.

Known issues

  • When integrating HTML controls in a webView via the NUSACustomVuiController, focus handling might not work correctly unless you are using iOS 6 or higher. [PSP00064357]
  • The SpokenText parameter in the CommandRecognized event is empty in command mode. [PSP00062085]
  • If you upgrade your MacOS X workstation from an version earlier than 10.7 to version 10.7 (or higher) dictation will not work on iOS 6 in the Simulator. Workaround: Install the workstation cleanly with MacOS X 10.7 (or higher). [PSP00063580]

Infrastructure no longer supported

  • Version 1.0 client-based applications will stop working
  • iOS 4.x and lower

Infrastructure changes planned for the next release

  • Support for iOS 5.x will be removed.
  • In 360 | SpeechAnywhere Services 1.5 all version 1.1 client-based applications will stop working.

System requirements

Recommended Supported
Operating System iOS 6.x iOS 5.x and higher
