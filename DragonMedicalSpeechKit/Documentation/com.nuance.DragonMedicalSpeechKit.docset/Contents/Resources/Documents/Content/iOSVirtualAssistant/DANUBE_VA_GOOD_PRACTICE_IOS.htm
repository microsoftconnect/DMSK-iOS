Good integration practices

To help users experience high recognition accuracy, follow these integration guidelines. Bad integration patterns can make it impossible for the system to adjust to the user's way of speaking and therefore decrease recognition accuracy.

Dynamic concepts

Dynamic concepts uploaded to the DMVA server via the uploadValuesForConcept method persist in DMVA after the session is closed; they remain available for the user until clearUploadedValuesForConcept is called. If a dynamic concept must not remain available after the DMVA session is closed, we recommended calling clearUploadedValuesForConcept before closing the session.

If values were uploaded for multiple concepts, you can call clearAllUploadedConceptValues instead of calling clearUploadedValuesForConcept for each concept.

If you need to update a concept for immediate use, call setInlineValuesForConcept instead of uploadValuesForConcept. When an inline concept value is no longer needed, remove it by calling clearInlineValuesForConcept.

Calling setInlineValuesForConcept to set a large number of dynamic concepts can affect performance. To set a large number of dynamic concepts, call uploadValuesForConcept at the beginning of the DMVA session.

Don't call uploadValuesForConcept and setInlineValuesForConcept on the same concept. If you've called uploadValuesForConcept on a concept and you need to update it with additional values, call uploadValuesForConcept on the concept again to upload the complete list of values to the DMVA server.

Error handling

Don't display the error codes returned by Dragon Medical Virtual Assistant to the user; they are useful to developers only.
