Speech-enabling your iOS app

Important: If you're deploying Dragon Medical Server at your site (and not hosted by Nuance), make sure to configure Dragon Medical SpeechKit for on-premises deployment before speech-enabling your app.

Prerequisites

You've created one or more GUI views in the Xcode's Interface Builder and the session has been initialized.

Procedure

For each GUI view to be speech-enabled, do the following:

  1. In Interface Builder, create a new object in your GUI view controller.
  2. Set its class to NUSAVuiController.

  3. Connect your GUI view to the view outlet of the NUSAVuiController object via drag-and-drop.

    If the view outlet of the NUSAVuiController object isn't visible in Interface Builder, add the NUSAVuiController.h header file, from the NuanceSpeechAnywhere\Headers folder, to your project.

  4. In Xcode, create a new outlet property (make sure you use the retain attribute) in your GUI view controller class and connect it to the NUSAVuiController object.

    5. The NUSAVuiController instance associated with the GUI view will become active whenever the GUI view is displayed and will handle all voice interaction with the GUI view.

Note: If you initialize the VuiController manually, add the initWithView method call in the viewWillAppear callback.

Your iOS app is now capable of speech recognition; all GUI views that are enabled and unhidden, whose editable and userInteractionEnabled properties are set to YES and for which speech recognition is supported, can be dictated into. For more information, see: Supported text controls.
