Error handling

To ensure that Dragon Medical SpeechKit is robust and fault-tolerant and that your app doesn't fail to work because of issues in Dragon Medical SpeechKit, the following error handling strategies have been developed:

  • Your integration will only be notified programmatically of errors that it can programmatically handle.
  • Errors related to networking and licensing that can't be corrected by the app won't be forwarded to the integration; instead end users will be informed that speech recognition is currently unavailable and will be asked to try again later.
  • In some cases, Dragon Medical SpeechKit performs validity checks on input values (for example, when creating an application command, a valid ID must be specified); however, in general it won't raise an exception if invalid input is provided or fail to work if a callback into integration code raises an exception. Dragon Medical SpeechKit will silently fall back to a stable state that allows speech recognition to proceed.
