Release Notes for 360 | SpeechAnywhere Services 2.1 Service Release 4 (iOS edition)

Solved issues in service release 4

  • The undo that voice command no longer moves the cursor to the wrong place. [DNB-4944]
  • Application commands defined using an escaped quotation mark (\") no longer cause an error. [DNB-4759]
  • Calling synchronizeWithView from the event handler for vuiControllerDidFinishProcessing or vuiControllerDidFinishProcessingForView no longer causes the application to stop responding. [DNB-4760]
  • The first speech-enabled control no longer automatically receives the UI focus when the VuiController is opened and synchronized. [DNB-5080]
  • libNuanceSpeechAnywhere.a no longer embeds CocoaLumberjack. [DNB-5026]
  • Text-To-Speech no longer causes a memory leak. [DNB-5044]

Important information

  • 360 | SpeechAnywhere Services 2.1 (iOS edition) is compatible with server versions 1.6, 2.0 and 2.1.
  • 360 | SpeechAnywhere Services 2.1 (iOS edition) will not work correctly when used with server version 1.5.
  • 360 | SpeechAnywhere Services 2.1 Server is compatible with client versions 1.5, 1.6, 2.0 and 2.1.
  • With Xcode 7/iOS 9 the bitcode option is enabled by default. The SpeechAnywhere SDK is not built with bitcode on; an existing SpeechAnywhere integration will not compile out of the box in Xcode 7. The Enable Bitcode option must be set to No manually by the integrator.
  • For on-premises deployments of SpeechAnywhere Servers with iOS 9, the connection to the SpeechAnywhere Server will only work if the App Transport Security requirements are met. Unencrypted HTTP connections to servers will no longer work.
  • Speech enabling text controls with the userInteractionEnabled or editable property set to NO is no longer supported.
  • SpeechAnywhere Analytics has been renamed to Dragon Medical Analytics.

New in version 2.1

  • The handling of temporary network problems has been improved.
  • Continuous recording across VUI form boundaries is now possible.
  • iOS 9 is now supported.
  • A SWIFT sample is now available for the iOS SDK.
  • The integrator can now actively improve speech recognition; notifying SpeechAnywhere about the document correction state as well as the finalized version via the support functionality for document workflows. For more information, see: 'Support functionality for document workflows' in the SpeechAnywhere SDK help.

Known issues

  • The SpokenText parameter in the CommandRecognized event is empty in command mode. [PSP00062085]
  • The speech bar is only displayed on the Xcode 6 Simulator if the soft keyboard is shown. Make sure the simulator is configured so that either the Simulator, Hardware, Keyboard, Connect Hardware Keyboard option is not selected, or the Toggle Software Keyboard option is also set. [PSP00077597]
  • A 16-byte memory leak occurs in the SDK each time a session is closed. [DNB-4846]
  • The iOS shortcut bar is not available when the speech bar is displayed. [DNB-5001]

Infrastructure

Infrastructure no longer supported

  • Version 2.1 client applications do not work with version 1.4 servers.

Infrastructure changes planned for the next release

  • In 360 | SpeechAnywhere Services 2.2 all version 1.5 and 1.4 client-based applications will stop working.
  • In 360 | SpeechAnywhere Services 2.2 all version 2.2 client-based applications will stop working with version 1.5 servers.

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third party product and supporting Nuance documentation.

When support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s end-of-life and Nuance's policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.
