Dragon Medical Virtual Assistant session management

The DMVA session can be in one of the following session states:

  • VAStateOpened
  • VAStateClosed

The methods openVA and closeVA are used to change the DMVA session state. The session state events are delivered to the vaSessionStateChanged callback.

openVA

To transition the session state from VAStateClosed to VAStateOpened, call openVA with the required parameters.

DMVA performs the required license validation before transitioning to the state VAStateOpened.

Note: The session state change is returned in the form of an event with the VAStateOpened state and the session ID assigned to that session. The session ID is provided for debugging purposes only; if an issue needs to be reported to Nuance, attach this session ID with other details.

closeVA

Call closeVA to close the active DMVA session. All pending transactions will be discarded.

Remarks

  • The SDK can also close the DMVA session because of network connectivity issues.

  • If the session is closed by the SDK, the app will get the event with the session state as VAStateClosed. The app will then be required to send a new openVA request to open a new DMVA session.
