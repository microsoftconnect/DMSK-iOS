Release Notes for 360 | SpeechAnywhere Services 1.6 Service Release 12 (iOS edition)

Solved issues in service release 12

360 | SpeechAnywhere Services occasionally shut down unexpectedly; this no longer occurs. [PSP00078461, PSP00078578]

The volume meter and sound quality indicators are now correctly displayed after rotating the device. [PSP00078701, PSP00078233]

The What You Can Say page is now correctly displayed in portrait and landscape modes. [PSP00078200]

You can now rotate the device while recording is on with no adverse effects. [PSP00078500]

Text can now be added to a speech-enabled control programatically, before it has the focus, without the Nuance flame covering the close icon. [PSP00078668]

Architecture armv7s is supported again. [PSP00078938]

Note: For information on previous service releases, see below.

Important information in version 1.6

  • 360 | SpeechAnywhere Services 1.6 Server is compatible with client versions 1.4, 1.5 and 1.6.
  • 360 | SpeechAnywhere Services 1.6 (iOS edition) is compatible with server versions 1.4, 1.5 and 1.6.
  • In iOS 7 applications microphone access must be permitted. If a user unintentionally disallows microphone access for an SAS-enabled application, this can be re-enabled; to enable microphone access, go to Settings, Privacy, Microphone. Note: The application is restarted automatically when this setting is changed.

  • From 360 | SpeechAnywhere Services 1.6 Service Release 7, applications integrating this SDK also have to link against libc++.dylib.

New in version 1.6

  • Voice editing functionality has been extended with select & say voice commands (Dragon-based systems only).
  • Text formatting has been improved. When text is inserted, the spacing is correctly updated and surrounding text is correctly capitalized (Dragon-based systems only).
  • SAS clients can now use Dragon customization packs (.dat files) containing Text-and-Graphics/auto-texts without lists. You can import and share .dat files via the Nuance Management Console.
  • The iOS edition now fully supports single line text fields.
  • The iOS edition now supports speech-enabled UISearchBar views.
  • The iOS edition has a new volume meter and audio quality indicator.
  • Improvements have been made to iOS edition applications when the network connection is unreliable or when switching between networks.
  • The iOS edition now supports 64-bit iOS-based applications.

Known issues

  • The SpokenText parameter in the CommandRecognized event is empty in command mode. [PSP00062085]
  • If you upgrade your Mac OS X workstation from a version earlier than 10.7 to version 10.7 (or higher) dictation will not work on iOS 6 in the Simulator. Workaround: Install the workstation cleanly with MacOS X 10.7 (or higher). [PSP00063580]
  • If startSpeaking is called again before the previous utterance is finished playing back, the new utterance will interrupt the playback of the previous one. Intended behavior is for the second call to be queued and started as soon as the first one has finished playing. [PSP00077479]
  • If a word is selected and the voice command add that to vocabulary is said, the Manage Vocabulary page is only partially displayed. Workaround: Do not select a word before opening the Manage Vocabulary page. [PSP00076910]
  • The speech bar is only displayed on the XCode 6 Simulator if the soft keyboard is allowed to be shown. Make sure the simulator is configured so that either the Simulator → Hardware → Keyboard → Connect Hardware Keyboard option is unchecked, or the Toggle Software Keyboard option is also set. [PSP00077597]

New in version 1.6 service release 7

360 | SpeechAnywhere Services now enables you to override the default sound quality indicators with custom images. To do this, set the volumeMeterTooLoudImage, volumeMeterTooSoftImage and volumeMeterTooNoisyImage properties of the NUSAVuiController and NUSACustomVuiController classes to the custom images. The images must be different for all three conditions, must not be blank and a minimum size of 40 by 40 pixels is required. It is recommended to use 150 by 150 pixel images on an iPad and 40 by 40 pixel images on an iPhone.

Note: Clear user feedback on sound quality is vital for the correct operation of 360 | SpeechAnywhere Services; use images that are clear to the user.

By default, when you tap into a speech-enabled field the keyboard stays hidden and only the speech bar is shown. A new property has been added so that you can change the behavior, and the keyboard is shown when you tap into a speech-enabled field. To change the default behavior, set the property: KeyboardVisibleByDefault in the NUSAVuiController class. [PSP00077404]
