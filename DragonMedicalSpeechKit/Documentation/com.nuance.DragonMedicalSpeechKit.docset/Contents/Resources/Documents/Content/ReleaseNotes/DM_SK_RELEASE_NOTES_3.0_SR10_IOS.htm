Release Notes for Dragon Medical SpeechKit 3.0 Service Release 10 (iOS edition)

Dragon Medical SpeechKit is a component of the Nuance Healthcare Development Platform. Rebranding the components is currently an ongoing activity. Dragon Medical SpeechKit was previously called 360 | SpeechAnywhere Services; please be aware that references to 360 | SpeechAnywhere Services 3.0 apply to this release.

Solved issues in service release 10

  • The volume is now correctly indicated on the GUI when users change how loudly they are speaking or change their position in relation to the microphone. [DNB-7313]

Solved issues in service release 6

  • Error recovery and support for load balancing have been improved. [DNB-6889]
  • The application no longer stops unexpectedly when using the Address Sanitizer. [DNB-6539]
  • Building an application for an older iOS SDK version no longer results in linker warnings. [DNB-6787]

New in service release 6

  • 360 | SpeechAnywhere Services client editions are now called Dragon Medical SpeechKit. [DNB-5670]
  • Dragon Medical SpeechKit now supports applications using bitcode. [DNB-4190]

Important information

  • Dragon Medical SpeechKit 3.0 (iOS edition) is compatible with server versions 2.0, 2.1 and 3.0.

  • Dragon Medical Server 3.0 is compatible with client versions 2.0, 2.1 and 3.0.

  • For on-premises deployments of Dragon Medical Server with iOS 9, the connection to Dragon Medical Server will only work if the App Transport Security requirements are met. Unencrypted HTTP connections to servers will no longer work.

  • Information on the voice commands changed with each version of Dragon Medical SpeechKit is now available in the end user help.

  • On the speech control/speech bar the colors for recording on/off have changed from red (on) - gray (off) to green (on) - red (off).

New in version 3.0

Manage Vocabulary (server version 3.0 required)

Dragon-based systems
  • Users can now search for words on the Manage Vocabulary page.
  • Users can now filter words on the Manage Vocabulary page based on the level (user/group/site) for which they were added.
  • New Modify Word screen: Users can now edit the spelling and pronunciation of words in their personal lexicon.
  • Users can now record how a word is pronounced.
  • New voice command to record how a word is pronounced: train word.

Personal auto-texts (server version 3.0 required)

All systems
  • Users can now create auto-texts via the personalization and help window, Manage Auto-texts page. This new functionality is available to desktop users and mobile device users with a Dragon Medical One license.
  • New voice command: make that an auto-text.

Speech recognition (server version 3.0 required)

Dragon-based systems
  • New English, Dutch, Flemish and German voice commands to form compound words. English examples: compound, compound that, compound selection.
  • New German voice commands to separate compound words: trenne das, Auswahl trennen, trenne Auswahl.
  • Improved correction alternatives for Dutch, French and German in the correction menu.
  • Dragon Medical now intelligently identifies sentence separators; this means voice commands for selecting sentences and navigating to sentences work more reliably.
All systems
  • New voice commands to navigate text: go to beginning of sentence, go to end of sentence, go to beginning of paragraph, go to end of paragraph.

On-premises systems (server version 3.0 required)

  • Dragon Medical systems now remove speech recognition data from the Dragon Medical File Share when the data is no longer needed for improving the recognition quality.

iOS edition

  • The iOS edition can now query if the Nuance Healthcare ID is enabled for the user.
  • The iOS edition can now open the personalization and help screen, showing specific tabs, via the integration.

Known issues

  • On iOS 9 devices, the application’s own online help is not displayed. [DNB-6213].

  • The SpokenText parameter in the CommandRecognized event is empty in command mode. [PSP00062085]
  • The speech bar is only displayed on the Xcode 6 Simulator if the soft keyboard is shown. Make sure the simulator is configured so that either the Simulator, Hardware, Keyboard, Connect Hardware Keyboard option is not selected, or the Toggle Software Keyboard option is also set. [PSP00077597]

  • A 16-byte memory leak occurs in the SDK each time a session is closed. [DNB-4846]

Infrastructure

Infrastructure no longer supported

  • Dragon Medical SpeechKit 3.0 applications do not work with version 1.5 servers.
  • In Dragon Medical Server 3.0 all version 1.4 client-based applications stop working.

Infrastructure changes planned for the next release

  • Dragon Medical SpeechKit 3.1 client applications will not work with version 1.6 servers.
  • In Dragon Medical Server 3.1 all version 1.5 and 1.6 client-based applications will stop working.
  • Dragon Medical SpeechKit 3.1 will stop support for iOS 7.

  • Dragon Medical SpeechKit 3.1 will stop support for Xcode 6 as a development environment.

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third party product and supporting Nuance documentation.

When support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s end-of-life and Nuance's policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.

Dragon Medical product life cycle

Nuance provides support throughout the life cycle of one version and point release. Please note that Nuance will not perform problem analysis after the end-of-life date stated here. If a Nuance-hosted system is used, Nuance cannot guarantee that an older version will continue working after the date stated here.
